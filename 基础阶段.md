## 第九阶段：迭代器与生成器

**目标：理解Python的迭代协议，掌握生成器的应用**

#### 可迭代对象与迭代器

可迭代对象（Iterable）：实现了`__iter__`方法的对象（list、dict、str等） 

迭代器（Iterator）：实现了`__iter__`和`__next__`方法的对象

for循环的本质：调用`__iter__`获取迭代器，然后不断调用`__next__`

```python
it = iter([1, 2, 3])

while True:
    try:
        x = next(it) # 每调用一次 next(it)，它就取出一个值，然后自动移动到下一个位置。
        print(x)
    except StopIteration:
        break
# for循环语法        
for 变量 in 可迭代对象:
    循环体
```

**一句话总结：可迭代对象 = “能被for循环遍历的东西”，迭代器 = “生产数据的工人”。**

#### 生成器

是可迭代对象

yield 关键字：把函数变成生成器。

```python
# 普通函数遇到return就结束
def normal():
    return 1
print("-" * 30)

def gen():
    yield 1 # yield 的意思是：先把这个值交出去，然后函数暂停在这里。下次再继续往下执行。
    yield 2 # yield返回的是一个生成器
    yield 3
g = gen()

print(next(g))  # 1
print(next(g))  # 2
print(next(g))  # 3    
```

惰性求值：用到一个才算一个，不会一次性把所有数据加载到内存	

生成器表达式：把列表推导式的`[]`换成`()`

iteratools 模块：强大的迭代器库工具，包含`chain、count、cycle、product、permutations、combinations`等

```python
from itertools import chain
from itertools import count
from itertools import cycle
from itertools import product, permutations, combinations

# chain把多个可迭代对象拼接起来，但是不是创建一个新列表，而是创建一个迭代器，用的时候一个个取
for x in chain([1, 2], [3, 4], [5]): 
    print(x)
# 无限计数器 count(start=0, step=1)
for x in count(1):
    print(x)
    if x == 5:
        break
# 循环重复
colors = cycle(["红", "绿", "蓝"])

print(next(colors))  # 红
print(next(colors))  # 绿
print(next(colors))  # 蓝
print(next(colors))  # 红

# 排列组合
print(list(product([1, 2], ["A", "B"]))) # 笛卡尔积
print(list(permutations([1, 2, 3], 2))) # permutations(可迭代对象, r) 从一组数据里选 r 个，考虑顺序。
print(list(combinations([1, 2, 3], 2))) # combinations(可迭代对象, r) 从一组数据里选 r 个，不考虑顺序。
```

一句话总结：生成器 = “按需生产”，处理大量数据时能省大量内存

浅拷贝：`copy.copy()`（只复制第一层，嵌套对象还是共享的）

深拷贝：`copy.deepcopy()` （完全独立的副本）

一句话总结：赋值不是复制，浅拷贝只复制第一层，深拷贝才是真正的“克隆”

踩坑：列表嵌套列表时，浅拷贝修改内层列表会影响原对象

#### 解包操作

序列解包：`a, b, c = [1, 2, 3]`

星号解包：`first, *rest = [1, 2, 3, 4, 5]`

字典解包：`{**dict1, **dict2}`

```python
d1 = {"a": 1, "b": 2}
d2 = {"b": 3, "c": 4}
merged = {**d1, **d2}   # 后面的键会覆盖前面的
print(merged)  # {'a':1, 'b':3, 'c':4}
```

函数参数解包：`func(*args, **kwargs)`

```python
def add(a, b, c):
    return a + b + c

nums = [1, 2, 3]
print(add(*nums))   # 列表解包

kwargs = {"a": 1, "b": 2, "c": 3}
print(add(**kwargs))  # 字典解包
```

一句话总结：解包就是把容器里的元素拆出来。Python 的解包可以用于列表、元组、**字符串**、字典以及函数参数传递。

#### 多线程与多进程（了解）

GIL （全局解释器锁）：Python多线程不能真正并行执行CPU密集型任务。在 CPython 解释器中，同一时刻通常只有一个线程能真正执行 Python 字节码。

threading 多线程：适合I/O密集型任务（网络请求、文件读写）

multiprocessing 多进程：适合CPU密集型任务（数据计算）

一句话总结：I/O密集用多线程，CPU密集用多进程

#### 协程基础（了解）

协程也是一种并发，但是和线程不同。线程是有操作系统调度的，而协程是由程序自己调度的。可以简单理解为：协程是在一个线程里，通过主动让出执行权，实现多个任务交替执行。协程也是适合I/O密集型任务。

async / awit 语法

asyncio 模块

异步编程的概念：单线程内的”并发“

一句话总结：协程是Python异步编程的基础，FastAPI就是基于协程的
